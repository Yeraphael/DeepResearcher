# DeepResearcher 项目快速启动手册

## 1. 项目结构

```text
DeepResearcher/
├─ backend/    # FastAPI + Python 后端
├─ frontend/   # Vue 3 + Vite 前端
├─ README.md
├─ 项目快速启动手册.md
└─ Bug解决.md
```

## 2. 环境要求

- Python `3.10+`
- Node.js `18+`
- npm
- `uv`

## 3. 后端启动

### 3.1 进入目录

```powershell
cd F:\Project\agent\DeepResearcher\backend
```

### 3.2 准备环境变量

```powershell
Copy-Item .env.example .env
```

后端会自动加载 [backend/.env](F:\Project\agent\DeepResearcher\backend\.env)。

推荐最小配置：

```env
DATABASE_PATH=./data/research.db
SEARCH_API=tavily
TAVILY_API_KEY=你的 tavily key

LLM_PROVIDER=custom
LLM_MODEL_ID=MiniMax-M2.1
LLM_API_KEY=你的真实 key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

说明：

- `DATABASE_PATH` 是研究历史持久化数据库路径，默认会落到 `backend/data/research.db`
- `SEARCH_API` 填后端名称，不是 API Key
- `LLM_MODEL_ID` 必须是平台真实模型名
- `LLM_API_KEY` 不能保留示例占位值

### 3.3 安装依赖

```powershell
uv sync
```

如果本机 `uv` 全局缓存有问题，可以临时指定：

```powershell
$env:UV_CACHE_DIR="F:\Project\agent\DeepResearcher\backend\.uv-cache"
uv sync
```

### 3.4 启动服务

```powershell
uv run python src/main.py
```

默认地址：

- 健康检查：[http://localhost:8000/healthz](http://localhost:8000/healthz)
- API 根地址：[http://localhost:8000](http://localhost:8000)

## 4. 前端启动

### 4.2 检查前端接口地址


## 5. 研究历史持久化说明

当前版本已经实现 SQLite 持久化：

- 左侧历史研究列表来自后端数据库，不再依赖前端内存
- 新建研究会创建新的 `research session`
- 点击左侧任意历史主题，右侧会重新加载该 session 的完整详情
- 服务重启后，历史研究仍可恢复

默认数据库文件：

- [backend/data/research.db](F:\Project\agent\DeepResearcher\backend\data\research.db)

主要表：

- `research_sessions`
- `research_tasks`
- `research_steps`
- `research_reports`
- `research_tool_calls`

## 6. Session API

当前研究历史相关接口：

- `GET /api/research/sessions`
- `GET /api/research/sessions/{id}`
- `POST /api/research/sessions`
- `POST /api/research/sessions/{id}/run`
- `POST /api/research/sessions/{id}/run/stream`

兼容旧接口：

- `POST /research`
- `POST /research/stream`

## 7. 推荐联调顺序

1. 启动后端
2. 访问 `GET /healthz`，确认返回 `{"status":"ok"}`
3. 启动前端
4. 点击“开始新研究”，创建一个 draft session
5. 输入主题并开始研究
6. 再点击“开始新研究”，确认旧主题仍保留在左侧
7. 重启后端和前端，确认左侧历史仍存在
8. 点击任意历史主题，确认右侧详情可恢复

## 8. 常见排查

### 8.1 401 `invalid_api_key`

说明：

- 这是外部模型平台鉴权失败，不是 session 持久化 bug

建议检查：

1. `LLM_API_KEY`
2. `LLM_BASE_URL`
3. `LLM_MODEL_ID`
4. 账号侧模型权限

### 8.2 404 `model not found`

说明：

- 当前 `LLM_MODEL_ID` 在目标平台不可用

建议检查：

1. 模型名是否拼写正确
2. 平台是否支持该模型
3. 当前账号是否已开通该模型

### 8.3 Tavily 配置错误

正确示例：

```env
SEARCH_API=tavily
TAVILY_API_KEY=tvly-xxxx
```

错误示例：

```env
SEARCH_API=tvly-xxxx
TAVILY_API_KEY=='tvly-xxxx'
```

### 8.4 DuckDuckGo `No results found`

说明：

- 这是搜索结果为空导致的真实运行失败
- session 仍会保留在历史列表中，可点击查看失败详情

建议：

1. 改用 `tavily`
2. 调整检索主题为更明确的英文或中英混合关键词
3. 通过左侧历史会话回看失败记录与错误信息
