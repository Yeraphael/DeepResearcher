# DeepResearcher 项目启动文档

本文档按你当前的本地环境来写：

- 操作系统：Windows
- Conda 环境：`deepresearch`
- 项目根目录：`F:\Project\agent\DeepResearcher`

如果你以后还是继续用这个 `conda` 环境，这份文档可以直接照着执行。

**一、启动前准备**
确保下面的软件已经安装好：

- Conda
- Python 3.10+
- Node.js 18+
- npm

项目目录结构：

```text
F:\Project\agent\DeepResearcher
├─ backend
├─ frontend
├─ README.md
└─ 项目快速启动手册.md
```

**二、第一次使用时怎么准备**
先打开一个新的 PowerShell，激活你的 conda 环境：

```powershell
conda activate deepresearch
```

进入后端目录：

```powershell
cd F:\Project\agent\DeepResearcher\backend
```

如果还没有 `.env`，先复制一份：

```powershell
Copy-Item .env.example .env
```

然后编辑 `backend\.env`，至少填这些：

```env
SEARCH_API=duckduckgo

LLM_PROVIDER=custom
LLM_MODEL_ID=你的模型名
LLM_API_KEY=你的真实密钥
LLM_BASE_URL=你的模型服务地址

HOST=0.0.0.0
PORT=8000
```

常见说明：

- `SEARCH_API` 填后端名，不是 API Key。
- 如果你用 `tavily`，还要额外填 `TAVILY_API_KEY=...`
- 如果你用 `perplexity`，还要额外填 `PERPLEXITY_API_KEY=...`
- 如果你本地跑的是 `ollama` 或 `lmstudio`，把 `LLM_PROVIDER` 改成对应值，并设置对应的 URL。

在 `deepresearch` 环境里安装后端依赖：

```powershell
cd F:\Project\agent\DeepResearcher\backend
python -m pip install -e .
```

安装前端依赖：

```powershell
cd F:\Project\agent\DeepResearcher\frontend
npm install
```

**三、日常启动方式**
推荐开两个终端窗口。

第 1 个终端启动后端：

```powershell
conda activate deepresearch
cd F:\Project\agent\DeepResearcher\backend
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
```

注意：

- 必须先 `cd` 到 `F:\Project\agent\DeepResearcher\backend`
- 不要在 `backend\src` 目录里启动
- 推荐命令是 `python -m uvicorn src.main:app --reload`
- 不要把启动目标写成 `main:app`，否则在包导入模式下容易出现模块导入问题

后端启动成功后，访问：

- 健康检查：[http://localhost:8000/healthz](http://localhost:8000/healthz)
- 接口文档：[http://localhost:8000/docs](http://localhost:8000/docs)

第 2 个终端启动前端：

```powershell
conda activate deepresearch
cd F:\Project\agent\DeepResearcher\frontend
npm run dev
```

前端默认访问地址：

- [http://localhost:5173](http://localhost:5173)

**八、常见问题排查**
1. `conda activate deepresearch` 失败

先确认环境存在：

```powershell
conda env list
```

如果没有 `deepresearch`，需要先创建你的 conda 环境。

2. 启动后端时报 `ModuleNotFoundError`

通常是下面几种原因：

- 当前没激活 `deepresearch`
- 后端依赖没装
- 没在 `backend` 目录启动
- 启动命令写错了

正确示例：

```powershell
conda activate deepresearch
cd F:\Project\agent\DeepResearcher\backend
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
```

错误示例：

```powershell
cd F:\Project\agent\DeepResearcher\backend\src
python -m uvicorn main:app --reload
```

这次修复过的典型报错就是：

```text
ModuleNotFoundError: No module named 'application'
```

如果你再次看到这类错误，先确认是不是没有按上面的“正确示例”启动。

重新执行：

```powershell
conda activate deepresearch
cd F:\Project\agent\DeepResearcher\backend
python -m pip install -e .
```

3. 启动前端时报 `npm` 相关错误

重新安装依赖：

```powershell
cd F:\Project\agent\DeepResearcher\frontend
rm -r node_modules
npm install
```

如果你在 PowerShell 下删目录不方便，可以用：

```powershell
Remove-Item -Recurse -Force node_modules
```

4. 后端报 `invalid_api_key`

说明模型服务或搜索服务密钥不对，重点检查：

- `backend\.env` 里的 `LLM_API_KEY`
- `backend\.env` 里的 `LLM_BASE_URL`
- `backend\.env` 里的 `LLM_MODEL_ID`
- 相关搜索服务的 API Key

5. 前端页面打开了，但研究请求失败

先按顺序检查：

- `http://localhost:8000/healthz` 是否正常
- `frontend\.env.local` 是否指向 `http://localhost:8000`
- 后端终端是否有异常报错

**十一、报告阶段超时怎么处理**
如果后端日志里看到类似下面的错误：

```text
hello_agents.core.exceptions.HelloAgentsException: LLM调用失败: Request timed out.
During task with name 'compile_report'
```

说明前面的规划、搜索、worker 摘要大概率已经完成，但最终报告撰写模型调用超时。

当前版本已经做了兜底处理：
- 报告提纲生成失败时，会自动使用代码生成的 fallback outline
- 最终报告 LLM 超时时，不再让 `/research/stream` 整条链路失败
- 系统会基于已完成的并行 worker 结果生成一份 Markdown 兜底报告
- 前端仍然会收到 `final_report` 和最后的 `done` 事件

这能保证研究流程不中断，但如果你想得到更完整、更像正式报告的结果，仍然建议检查：
- `backend\.env` 里的 `LLM_BASE_URL` 是否稳定
- `LLM_MODEL_ID` 对应的模型是否响应较慢
- 当前研究主题是否过大，导致最终报告上下文过长
- 本地模型服务是否已经加载好模型
